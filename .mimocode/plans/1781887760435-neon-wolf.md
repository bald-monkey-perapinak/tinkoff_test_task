## Блок 1: Прокси и обход блокировки hh.ru

**Проблема:** hh.ru возвращает 403 Forbidden — блокирует нестандартный User-Agent и/или IP.

### 1.1 Обход без прокси (приоритет)
- **Файл:** `backend/services/hh_client.py`
- Заменить User-Agent на реалистичный (например `Mozilla/5.0 (compatible; VacancyAgent/1.0)`)
- Добавить заголовки `Accept: application/json`, `Accept-Language: ru-RU,ru;q=0.9`
- Добавить поддержку `HH_PROXY` из env (http/https/socks5)
- Если прокси не задан — работать напрямую с улучшенными заголовками

### 1.2 Прокси с ротацией
- **Файл:** `backend/config.py` — добавить `HH_PROXY` и `HH_PROXY_LIST`
- **Файл:** `backend/services/hh_client.py` — если задан `HH_PROXY_LIST` (путь к файлу), читать список прокси и ротировать при ошибках
- **Файл:** `.env.example` — добавить `HH_PROXY=http://host:port` и `HH_PROXY_LIST=proxies.txt`
- Добавить `httpx[socks]` в requirements.txt для SOCKS5 поддержки

### 1.3 Fallback на mock-данные
- **Файл:** `backend/services/hh_client.py`
- Если hh.ru недоступен после всех retry — загружать вакансии из `data/sample_vacancies.json`
- Логировать что используется fallback

## Блок 4: Кеширование LLM-результатов

**Проблема:** Каждый вызов /api/analyze — это новый Groq API call. Повторный анализ тех же вакансий с теми же критериями стоит денег.

### 4.1 Backend
- **Файл:** `backend/database.py` — новая таблица `analysis_cache` (key TEXT PRIMARY KEY, results_json TEXT, report TEXT, created_at REAL)
- **Файл:** `backend/services/analyzer.py` — перед вызовом LLM проверять кеш по хэшу (vacancy_ids + criteria). TTL = 1 час
- **Файл:** `backend/main.py` — перед analyze проверять кеш, возвращать из кеша если есть

## Верификация

1. **Прокси:** Запустить `uvicorn backend.main:app`, вызвать `GET /api/search?query=python` — должен вернуть 200 с вакансиями
2. **Пагинация:** Поиск → scrolling → "Загрузить ещё" → новые вакансии добавляются
3. **Экспорт:** `GET /api/export?format=csv` → корректный CSV
4. **Кеш:** Два одинаковых запроса analyze → второй должен быть быстрым (из кеша)
5. **Health:** `GET /api/health` → JSON со статусом
6. **Тесты:** `pytest backend/tests/ -v` — все 78+ тестов зелёные
