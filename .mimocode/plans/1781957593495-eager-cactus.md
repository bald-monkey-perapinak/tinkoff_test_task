## Пункт 2. SUBMISSION.md

**Файл:** `SUBMISSION.md` (в корне репозитория)

Структура (5-7 предложений + буллеты):

```markdown
# Submission Summary

## Что решает
Агент для поиска и предварительного отбора junior-вакансий/стажировок —
автоматизирует ручной скролл hh.ru, фильтрует по критериям и выдаёт
 ранжированный отчёт с объяснениями.

## Как устроен pipeline
hh.ru API (публичный, без авторизации) → парсинг (JSON/CSV) →
LLM-анализ (Groq, Llama 3.3 70B) → markdown-отчёт.
Параллельно: Telegram Mini App для визуала + Telegram Bot для
уведомлений о новых вакансиях.

## Где LLM, а где обычная логика
LLM используется ТОЛЬКО в `analyze_with_llm` для ранжирования и
объяснений. Всё остальное — детерминированная логика: парсинг,
генерация отчёта, поиск по API, уведомления. При отсутствии API-ключа
работает rule-based fallback.

## Ограничения и риски
- Бесплатный tier Groq (rate limit ~30 req/min)
- hh.ru может блокировать по IP при высокой нагрузке
  (отсюда proxy-логика + mock-fallback)
- SQLite — для прототипа, не для продакшена

## Что было сложнее всего
Балансировка между надёжностью интеграции с hh.ru (retry, proxy,
mock-fallback) и простотой кода. А также sanitization промпта для
LLM — защита от injection без потери полезного контента.
```

## Пункт 4. Вынести сильные места в README

**Файл:** `README.md` — новый раздел "Надёжность и безопасность" перед "Ограничения"

```markdown
## Надёжность и безопасность

- **Prompt sanitization** (`_sanitize()` в analyzer.py) — защита от
  injection в LLM-промпт, фильтрация опасных символов (<, >, |, \, `)
- **Retry с exponential backoff** — на Groq API (3 попытки, задержка 1→2→4с)
  и на hh.ru API (3 попытки + respect Retry-After при 429)
- **Кэширование анализа** — SHA-256 hash от (вакансии + критерии),
  повторный запрос отдаёт кэш без обращения к LLM
- **Rule-based fallback** — при отсутствии Groq API-ключа анализ
  работает на детерминированных правилах (направление, зарплата,
  навыки, формат)
- **Mock-fallback для hh.ru** — при недоступности API загружаются
  sample-данные из `data/sample_vacancies.json`
```
